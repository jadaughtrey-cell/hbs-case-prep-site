Setup Guide

Mirror It On Your Machine

Full setup takes 15–20 minutes. The three hardest parts are Canvas token (3 min), HBSP login (5 min), and course config (5 min). Everything else is copy-paste. You run it from your own machine against your own accounts — nothing is stored externally.

3

Canvas API Token

A personal access token lets the scraper read your assignments via the Canvas REST API — no browser window, no clicking through pages. This takes about 3 minutes.

1

Go directly to your Canvas profile settings

Open this URL in your browser — it works for any Canvas school:

https://hbs.instructure.com/profile/settings
↑ replace "hbs" with your school's Canvas subdomain if different
2

Scroll all the way to the bottom → find "Approved Integrations"

It's at the very bottom of the Settings page. Click + New Access Token.

ℹ️

In the dialog: Purpose = Case Prep Script  ·  Expiry = leave blank (permanent)  ·  click Generate Token.

3

Copy the token immediately — Canvas only shows it once

It starts with ~ or a long alphanumeric string. Store it somewhere secure before closing the dialog.

4

Set it as an environment variable (don't hardcode in source)

# Windows PowerShell — persists across reboots:
> [System.Environment]::SetEnvironmentVariable("HBS_CANVAS_TOKEN","your_token","User")
# Then restart your terminal. Verify it worked:
> $env:HBS_CANVAS_TOKEN
your_token_here ← should print your token
# Mac/Linux — add to ~/.zshrc or ~/.bashrc:
$ echo 'export HBS_CANVAS_TOKEN="your_token"' >> ~/.zshrc && source ~/.zshrc
⚠️

Canvas tokens grant full account access (read/write). Never commit this to git. If you accidentally expose it, delete it from Canvas Settings → Approved Integrations immediately and generate a new one.

4

HBSP Authentication

HBSP (Harvard Business School Publishing) case PDFs are behind a paywall linked to your Canvas account — you can't just download them with a URL. This step opens a real browser, you log in once, and the script saves your session so all future downloads happen silently in the background.

1

Run the setup command — it opens a Chromium window

$ python hbsp_downloader.py --setup
Opening Chromium browser window — do not close it yet.
2

In the browser window that opens: log into Canvas normally

Go to hbs.instructure.com and sign in with your HBS credentials. This is the same login you use every day — just do it in this specific Chromium window, not your regular browser.

3

Navigate to any Canvas assignment that has a case PDF link

Open one of your courses → go to any assignment that has a case reading. Look for a link that says something like "Harvard Business School Case" or opens a PDF on hbsp.harvard.edu. Click it and wait for the PDF to load fully. This is what establishes the HBSP session.

⚠️

The PDF must actually load in the browser. If you see a "purchase required" page, make sure your Canvas course has an active coursepack. If the PDF loads in a preview pane, that counts — you don't need to open a separate tab.

4

Close the browser window — the script saves your session automatically

Saving session to hbsp_auth.json ...
✓ 42 HBSP cookies · 18 Canvas cookies confirmed.
✓ Ready to download. Sessions last ~30 days.
ℹ️

hbsp_auth.json stores two things: HBSP session cookies (for case PDFs) and Canvas session cookies (for xlsx/docx files uploaded directly to Canvas assignments — not linked via HBSP). Both expire after ~30 days. When downloads start returning HTML instead of PDF, that's your signal to re-run --setup.

Anthropic API Key

Used to call Claude Sonnet for case analysis. Each case costs roughly $0.05–0.15 depending on length.

1

Get your key

Go to console.anthropic.com → API Keys → Create Key. Copy it — it's only shown once.

2

Set the environment variable

# Windows PowerShell:
$ [System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY","sk-ant-...","User")
# Mac/Linux:
$ export ANTHROPIC_API_KEY="sk-ant-..."

Course Configuration

Edit class_prep_workflow_v2.py to point each course at your Canvas course ID and local folder.

# class_prep_workflow_v2.py — edit COURSES dict

BASE = os.path.join(os.path.expanduser("~"),
    "OneDrive", "Documents", "YourSchool", "Courses")

COURSES = {
    "FIN2": {
        "canvas_course_id": 16192,      # from your Canvas URL
        "canvas_url": "https://hbs.instructure.com/courses/16192/assignments",
        "folder": os.path.join(BASE, "Finance"),
        "accent": "1B4F72",             # hex color for cheatsheet headers
        "light":  "EBF4FA",
        "label":  "FIN2",
        "class_start": "10:40",         # fallback if Graph API unavailable
        "class_end":   "12:00",
    },
    "BGIE": { ... },
    "DSAIL": { ... },
}
ℹ️

Finding your Canvas course ID: Go to your course in Canvas. The URL will be canvas.youruniversity.edu/courses/12345/assignments — that number is your canvas_course_id.

Config reference

KeyDescriptionRequired?
canvas_course_idInteger from the Canvas URL for this courseYes
canvas_urlFull URL to the assignments list page (used by browser-auth flow)Yes
folderLocal path where downloaded files and cheatsheets are savedYes
accent / lightHex colors used for cheatsheet section headersOptional
labelShort display name (used in Outlook event names)Optional
class_start / class_endFallback times (HH:MM 24h) if Graph API unavailableOptional

Microsoft Graph API Optional

This step is what allows the pipeline to rename and attach files to the HBS-pushed calendar events like "FIN2 C". Without it, the fallback MAPI COM sync can only modify calendar items you own — not the ones HBS pushes. Skip this step if you're just testing cheatsheet generation — you can add it later. If you're on a corporate network with strict OAuth policies, you may need to request an app registration from your IT team.

1

Install MSAL

$ pip install msal
2

Authenticate (device code flow — one-time)

$ python graph_auth.py
Visit: https://microsoft.com/devicelogin
Enter code: ABCD-1234
Sign in with your Microsoft account...
✓ Token cached. Graph API ready.

First Run

Run the full pipeline for a date range. Use flags to skip stages or target specific courses.

weekly_prep.py
# Run everything for the coming week:
$ python weekly_prep.py 2026-04-14 2026-04-17
# Skip file downloads (cheatsheets only):
$ python weekly_prep.py 2026-04-14 2026-04-17 --skip-download
# Target specific courses:
$ python weekly_prep.py 2026-04-14 2026-04-17 --courses BGIE FIN2
# Skip Outlook sync (generate cheatsheets only):
$ python weekly_prep.py 2026-04-14 2026-04-17 --skip-outlook
# Use more powerful model for complex cases:
$ python weekly_prep.py 2026-04-14 2026-04-17 --model claude-opus-4-6

Troubleshooting

⚠️ "anthropic SDK is required" error

The SDK cannot be installed in some sandboxed environments. The pipeline uses a direct requests.post() call to the Anthropic API as a fallback — no SDK needed. If you see this error, ensure ANTHROPIC_API_KEY is set in your environment.

⚠️ Blank or empty .docx files

Caused by the API response being cut off before the JSON was complete (max_tokens too low). Fix: re-run with --depth detailed, which uses a higher token budget. The default intermediate depth has 5,000 token budget — sufficient for 90% of cases.

⚠️ HBSP downloads failing with HTML response

Your Canvas session has expired. Re-run python hbsp_downloader.py --setup to refresh the auth cookies. Sessions last approximately 30 days.

⚠️ Outlook event not found ("not_found")

The Graph API can't find an event for that course/date. Verify the course abbreviation in COURSES matches the first word of your Outlook event (e.g. "FIN2" matches "FIN2 C"). Run python outlook_sync.py --list 2026-04-15 to see all events Outlook has on that date.

⚠️ Canvas scrape returns 0 assignments

Check that HBS_CANVAS_TOKEN is set and valid (echo $env:HBS_CANVAS_TOKEN). Also verify your canvas_course_id matches the 5-digit number in the Canvas URL. The API uses a bucket=future filter — if running for past weeks, remove it in canvas_scraper.py line ~480.

⚠️ SSL certificate error ("CERTIFICATE_VERIFY_FAILED")

Common on school or corporate networks that use a proxy with a self-signed certificate. The pipeline's API call uses verify=False as a fallback, which suppresses this. If you still see it, make sure you're running the latest version of ai_case_analyzer.py and that requests is up to date (pip install -U requests).

⚠️ Files downloading as 0 bytes or wrong type

The downloader uses magic bytes (actual file content) rather than the server's Content-Type header to detect file types. If a file comes back as 0 bytes, the Canvas session probably expired — re-run hbsp_downloader.py --setup. If the file is named .unknown, the magic bytes didn't match any known type — check the raw bytes with xxd filename.unknown | head to diagnose.

✅ Pre-Flight Checklist

  • Python 3.11+ installed and accessible from terminal
  • All dependencies installed from requirements.txt
  • Playwright Chromium installed (python -m playwright install chromium)
  • HBS_CANVAS_TOKEN environment variable set
  • HBSP auth session saved (hbsp_auth.json exists)
  • ANTHROPIC_API_KEY environment variable set
  • COURSES dict in class_prep_workflow_v2.py updated with your course IDs and folder paths
  • Target folders exist on disk (pipeline will not create them)
  • (Optional) Graph token cached via python graph_auth.py
