The problem

Not a knowledge problem. A volume problem.

The issue isn't that summaries are hard — it's that you need to do the exact same work over and over, for every single class, until the semester ends.

Pain 01

The same steps, every single week

Log in. Find assignment. Find case link. Download PDF. Find exhibit. Download that too. Rename. Repeat — for 10+ classes, every week, 15 weeks in a row.

Pain 02

ChatGPT works once. Not at scale.

Pasting a PDF into ChatGPT is free and fast. Doing it 10 times a week, formatting the output, saving it somewhere useful, finding it again before class — that's hours. Every week.

Pain 03

Your calendar tells you nothing

HBS pushes events named "FIN2 C" with no body, no attachments, no context. You check your calendar Sunday night and have no idea which case you're walking into Tuesday.

Pain 04

Files end up in the wrong places

PDFs in Downloads. Exhibit in another tab. Notes in a third place. By the time class starts you're still hunting for the spreadsheet that everyone else already has open.

How it works

Five stages. One command. Nothing left to do.

The pipeline runs against every class in the date range — scraping, downloading, analyzing, building, and syncing without any manual steps between stages.

Cheatsheet scales to case length

Output is proportional, never padded.

Word count is calibrated from page count before the API call — short cases get concise summaries, long cases get detailed analyses. Hard cap at 2,000 words.

4-page case~500 words
10-page case~900 words
16-page case~1,300 words
22-page case~1,800 words
38+ pages (hard cap)~2,000 words
Each cheatsheet includes: executive summary · key timeline · actor map · argument landscape (3-column) · hidden assumptions · per-question Q&A answers with citations.

What's inside

Built for the exact HBS workflow.

Six components, each handling a specific part of the problem — connected into one runnable pipeline.

🎓

Canvas REST API

Reads assignments, due dates, embedded questions, and file links using the Canvas API — structured data, not browser scraping.

canvas_scraper.py
🔐

HBSP + Canvas Auth

One-time Playwright login saves a session file. All future HBSP case PDF downloads and Canvas-native file downloads run silently with no browser windows.

hbsp_downloader.py
📂

Any File Type

Downloads PDFs, Excel exhibits, Canvas-native uploads, and external links. File type is detected from magic bytes — not filename or Content-Type header.

hbsp_downloader.py
🤖

Claude Sonnet Analysis

Full case text plus assignment questions sent to claude-sonnet-4-6. Returns structured JSON calibrated to case length — timeline, argument framework, Q&A, hidden assumptions.

ai_case_analyzer.py
📝

Word Doc Builder

JSON analysis → formatted .docx using python-docx. Metadata tables, section headers, argument comparison tables, bullet Q&A blocks with course-color accents.

gen_cheatsheets_v2.py
📅

Outlook Calendar Sync

Microsoft Graph API finds HBS push events by course code, renames them with the full case title, attaches every downloaded file, and embeds video links in the event body.

outlook_sync.py
